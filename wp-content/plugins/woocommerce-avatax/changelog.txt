2026.07.14 - version 3.8.5
* Feature - Support for Malaysia MY-B2B-EINVOICE e-invoicing mandate for invoices and credit notes.
* Feature - Support for Singapore SG-B2B-PEPPOL and SG-B2B-REPORTING e-invoicing mandates for invoices.
2026.06.29 - version 3.8.4
* Feature - Expanded France e-invoicing support with application response polling, payment reporting, and enhanced B2B/B2C e-reporting through Avalara E-Invoicing and Live Reporting.
2026.06.15 - version 3.8.3
* Feature - Added support for Section 232 metal attributes, allowing product-level metal composition and country-of-origin data to be captured and automatically included in tax requests for eligible U.S. inbound transactions.
2026.05.26 - version 3.8.2
* Feature - Added credit note support for the France FR-B2B-EINVOICE E-Invoicing and Live Reporting mandate, extending the existing invoice functionality.
* Feature - Added support for the Romania RO-B2B-EINVOICE E-Invoicing and Live Reporting mandate for order and refund transactions.
* Fix - Improved address validation by enabling U.S. and Canada validation independently of EU nexus settings and applying validation rules based on the shipping country.
2026.04.28 - version 3.8.1
* Feature - Added support for Poland’s PL-B2B-ENVOICE e-invoicing and live reporting mandates for orders and refunds, and the plugin package includes required seller-level and buyer-level custom fields plus the standard Poland field-mapping template by default.
* Feature - Added a connector option to select shipment type as postal service for flat rate shipments, and WooCommerce now supports customs duty calculation for U.S. inbound transactions using postal-based rules instead of the default commercial method.
* Feature - Added credit note support for France’s FR-B2B-EINVOICE mandate, and the plugin package includes the standard France field-mapping template by default. You can use the default mapping or customize it to fit your business needs.
* Fix - Fixed an issue where the plugin accessed the cart before WordPress and WooCommerce finished initializing, which caused repeated PHP warnings in flows like cart, checkout, and address validation, especially with Blocks-based checkout.
* Fix - Fixed the plugin to work reliably with Blocks-based and early-initializing checkout flows like funnel or phone orders, where it previously tried to read cart data before WooCommerce fully initialized the cart and session, causing warnings and occasional checkout issues.
* Fix - Fixed an issue where WooCommerce showed the internal exemption flag is_line_exempted: 1 in customer emails and order details because the plugin stored it as visible metadata and only hid it in admin views.
* Fix - Fixed an issue where enabling AvaTax for WooCommerce caused Easyship to disappear from shipping settings and zones and triggered errors that prevented shipping methods from loading.
2026.03.24 - version 3.8.0
* Feature - Added support for Italy's IT-B2B-EINVOICE and IT-B2G-EINVOICE e-invoicing and live reporting mandates.
* Feature - Added WooCommerce - Avalara reconciliation feature.
* Fix - Fixed an issue where order is incorrectly marked as sent to AvaTax when origin address is invalid.
2026.03.10 - version 3.7.1
* Feature - Added a new Update Address option in the WooCommerce connector Tools section that lets you sync and immediately apply the correct default company address to all new transactions sent to AvaTax. It fetches the updated default company location from AvaTax and sets it as the origin address in the connector.
* Fix - Fixed a performance issue with loading admin pages in WooCommerce where ELR-related admin scripts loaded and executed even when HPOS was disabled and ELR was not connected, resulting in delay in loading the admin pages.
* Fix - Fixed an issue that caused checkout to fail when two subscription products were added to the cart where the second subscription product lost the required subscription metadata, including the subscription interval and period. This fix ensures the correct subscription interval and period data is preserved for each item.
* Fix - Fixed an issue where WooCommerce invoices displayed tax as applied even though AvaTax calculated the transaction as exempt in the Avalara portal. The issue occurred especially with classic checkout for guest customers where the checkout page sent Guest instead of the customer email address or customer code, creating a discrepancy between WooCommerce and the Avalara portal.

* Feature - Added configurable certificate validation to let customers choose between auto-validation and queue submission.
* Fix - Updated the connector to send taxes when a WooCommerce order reaches the Completed status, even if intermediate order statuses are skipped.
